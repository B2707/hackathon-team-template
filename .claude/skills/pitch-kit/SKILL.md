# pitch-kit

Knowledge pack consumed by the `pitchdeck`, `judge-simulator`, and
`submission-prep` skills and the `/pitch` and `/judge-sim` commands.

## Files

| File | Use |
|---|---|
| `knowledge/judging-criteria.md` | Standard judging dimensions + typical weights (the event rubric overrides these) |
| `knowledge/winning-patterns.md` | Ten patterns consistently seen in winning teams |
| `knowledge/demo-psychology.md` | How judges perceive demos, and techniques that exploit it |
| `knowledge/pitch-strategy.md` | Narrative structure and delivery strategy |
| `knowledge/submission-guidelines.md` | Platform submission requirements and checklist |
| `templates/pitchdeck-outline.md` | Seven-slide deck skeleton |

## Provenance

Curated from `bernieweb3/hackathon-ai-devkit` @ `260b4295` (MIT), security
reviewed and trimmed per team decision #74. Q&A preparation lives in the
`judge-simulator` skill; the upstream "push direct to main" git pattern was
removed — this repo is PR-only by design.
